# Cover-letter Templates

자기소개서 (cover-letter) 작성 시 writer subagent 가 로드. STAR 기반 + 한국 채용 시장 관습 반영.

## 기본 구조 (5 섹션)

```markdown
# [지원 직무 / 회사] - [지원자 한 줄 요약]

## 1. 지원 동기와 핵심 단언
[가장 단단한 한 단언. 회사 + 직무 + 본인의 fit 을 한 문장에. 수치 1개 이상.]

## 2. 경험 1: [핵심 사례]
- **Situation**: [문제 상황 + 수치]
- **Task**: [본인이 맡은 역할]
- **Action**: [구체 조치 - 코드/설계/조정]
- **Result**: [측정 가능한 결과]

## 3. 경험 2: [보완 사례]
[같은 STAR 패턴, 다른 각도]

## 4. 직무 적합성
[지원 회사의 어떤 도메인/제품에 어떤 자산이 즉시 적용되는가]

## 5. 마무리
[면접 기회 요청. 한 줄. 추가 자료 옵션 1줄.]
```

## STAR 작성 원칙

| 요소 | 작성 원칙 | 예시 |
|------|----------|------|
| Situation | 일반화 회피, 구체 수치 | "월 결제 처리량 10억원, 정산 지연 평균 2시간" |
| Task | 본인 책임 영역만 | "정산 배치 retry 정책 재설계 단독 담당" |
| Action | 명령형 동사, 측정 가능한 행동 | "exponential backoff (max 5회) + DLQ 도입" |
| Result | 수치 비교, before/after | "재시도 자동화율 40% → 92%, 운영자 수동 개입 0건" |

## 톤 가이드

- 기본 `~습니다` 체. 격식 우선.
- 첫 문장은 단언으로. "안녕하세요" 등 인사말 금지.
- 자랑형 형용사 0회 - "열정적으로 임했습니다", "최선을 다해 노력했습니다" 같은 추상 수식 금지.
- 1인칭 "저" 명시 적당히 - 한 단락에 1회 이내.

## 첫 문장 (opening) 패턴

좋은 opening 은 가장 단단한 단언. 4가지 패턴 중 1개:

1. **수치 단언**: "현재 회사에서 결제 정합성 사고 0건을 18개월간 유지하고 있습니다."
2. **자산 매핑**: "지원 회사의 송금 정합성 문제는, 제가 saga 패턴으로 직접 해결한 결제 정합성 문제와 같은 구조입니다."
3. **결과 우선**: "ArgoCD 표준화로 배포 실패율을 8% 에서 0.4% 로 떨어뜨렸습니다."
4. **전환 선언**: "프론트엔드 2년차로서, design system 도입과 성능 회귀 모니터링 두 축에 동시 기여하고자 지원합니다."

피할 opening:

- "어릴 적부터 ~ 에 관심이 많았습니다" - 진부함
- "저는 ~ 입니다" 단순 자기소개 - 정보 가치 0
- "귀사의 ~ 에 깊은 인상을 받았습니다" - 검증 불가 칭찬

## 마무리 (closing) 패턴

3-4 가지 짧은 closing 중 1개:

1. **자료 옵션**: "PR 일부 / Storybook / 실험 노트북은 GitHub 링크로 공개 가능합니다. 면접 기회를 요청드립니다."
2. **검증 가능 회수**: "지원 직무에 대해 30분 통화로도 위 자산을 검증해 드릴 수 있습니다."
3. **단정 마무리**: "면접 자리에서 직접 검증해 보고 싶습니다."

피할 closing:

- "긴 글 읽어주셔서 감사합니다" - filler.
- "꼭 채용해 주십시오" - 구걸.
- "최선을 다하겠습니다" - 추상.

## 본문 길이 가이드

- 신입/주니어: 800-1200자
- 3-5년차: 1200-1800자
- 시니어: 1800-2500자

길수록 좋은 게 아니라 STAR 한 사이클이 완결되어야 좋다. 사례 부족하면 짧게.

## 금기 (cover-letter 전용)

- "사회적 책임", "글로벌 인재", "조직 발전에 기여" 류 슬로건 - 모두 0회.
- 회사 historical 칭찬 ("국내 1위 ~") - 사정관도 알고 있음, 정보 가치 없음.
- 이력서와 동어 반복 - cover-letter 는 이력서가 못 하는 "왜 이 회사" 를 답해야 한다.

## 검증 체크리스트 (writer 가 마지막에 self-check)

- [ ] 첫 문장이 단언인가? (질문/인사 아님)
- [ ] 각 STAR 사례에 수치 1개 이상?
- [ ] 회사명/직무명 본문 1회 이상 명시?
- [ ] 추상 수식어 (열정/최선/꾸준히) 0회?
- [ ] 톤 일관성 (~습니다 또는 ~다 한쪽)?
- [ ] em-dash/en-dash 본문 0회 (ai-tell-rules.md R7)?
