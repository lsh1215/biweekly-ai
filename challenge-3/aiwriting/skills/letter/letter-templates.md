# Letter Templates

letter-writer 가 로드. 한국어 개인 편지 (감사 / 축하 / 위로 / 추천) 작성 시 적용.

## 핵심 명제

좋은 letter 의 본질은 **구체적 한 장면**. 일반화한 칭찬·위로는 기계가 쓴다. 사람은 그 사람의 한 순간을 짚는다.

## 1. 감사 편지

```markdown
## opening
[근황 한 줄. 받는 사람과 본인의 현재 거리감을 짧게.]

## body1
[받은 가르침/도움 중 한 가지를 짚는다. 한 줄/한 장면.]
[그것이 지금 어떻게 쓰이고 있는지 구체 사례 1개.]

## (body2 optional)
[필요하면 한 사례 더. 아니면 생략.]

## closing
[다시 만날 가벼운 제안. 시간 압박 없이.]
```

피할 패턴:

- "정말 감사드립니다" 만 반복 - 감정어가 사례를 못 이긴다.
- "잊지 않겠습니다" - 추상.
- "OO님 덕분에" 만 반복 - 한 번이면 충분.

## 2. 축하 편지

```markdown
## opening
[축하 인사. 받는 사람의 시간 (결혼식/승진/출산/입학 등) 명시.]

## body1
[축하의 자리에서 본 한 장면. 그 장면이 받는 사람의 본 모습이라는 단언.]

## (body2 optional)
[그 본 모습이 앞으로 어떻게 이어질지 짧은 믿음 표현.]

## closing
[다음 만남 제안. 가벼운 톤.]
```

피할 패턴:

- "행복하시길 바랍니다" 만 반복 - 진부함.
- 결혼 축하인데 신부/신랑 한쪽만 언급.
- 일반론 ("결혼은 인생의 큰 전환점이며 ~") - 누구에게나 맞는 문장.

## 3. 위로 편지

```markdown
## opening
[정형구 OK 1회 - "삼가 고인의 명복을 빕니다" / "마음 깊이 위로드립니다"]

## body1
[위로의 말 자체가 한계 있음을 솔직히 인정하는 한 줄.]
[받는 사람의 감정 무게를 인정.]

## body2
[할 수 있는 작은 도움 하나를 구체적으로 제안.
 예: 업무 인수인계, 회의 대행, 식사 한 끼. 부담 없이 받을 수 있는 형태로.]

## closing
[건강 챙기시기를 / 마음 추슬릴 시간 가지시기를. 짧게.]
```

피할 패턴:

- "힘내세요" - 명령. 위로가 아님.
- "시간이 약입니다" - 진부함 + 가르침 톤.
- "이해합니다" - 검증 불가, 위계.
- 종교적 표현 - 받는 사람 종교 모르면 0회.

## 4. 추천서

```markdown
## opening
[추천인 본인 소개 + 학생/지원자와의 관계 + 함께한 기간.]

## body1
[강점 - **단 한 가지** 명제로 압축. 그 명제를 뒷받침하는 구체 사례 1개.
 수치 가능하면 수치 (예: "0.41 → 0.69").]

## body2
[약점 / 보완 필요 영역도 솔직히 - intellectual honesty.
 단, 약점이 추천 결정을 흔들지 않을 수준임을 명시.]

## body3
[그 사람이 graduate/지원 환경에 적합한 이유.
 강점이 그 환경의 어떤 요구와 매칭되는지.]

## closing
[필요시 추가 자료/통화 가능. 한 줄.]
```

피할 패턴:

- 약점 없음 - reviewer 가 가장 의심하는 추천서.
- "매우 우수한 학생입니다" 만 반복 - 추상 형용사.
- 9년치 칭찬 나열 - 한 가지 명제로 압축이 더 강하다.

## 톤 매트릭스

| 관계 | 톤 | 비고 |
|------|-----|------|
| 격식 (멘토, 상사, 선생님, 사정관) | `~습니다` | 호칭 명확히 |
| 친밀 (10년 친구, 형제) | `~다` | 자연스러운 구어체 |
| 위로 letter | `~습니다` | 친밀해도 격식 유지 |
| 직장 동료 (친한 사이 X) | `~습니다` | 거리감 유지 |
| 가족 | `~다` 또는 `~요` | 가족 호칭 추가 |

## 길이 가이드

- 감사: 200-400자 (짧을수록 진심 전달)
- 축하: 200-400자
- 위로: 150-300자 (짧은 게 위로)
- 추천서: 800-1500자 (격식상 더 길게)

## 검증 체크리스트 (writer 가 마지막에 self-check)

- [ ] 수신자가 본문 1회 이상 호명?
- [ ] 구체적 한 장면/사례가 본문에 있는가?
- [ ] 일반론으로 시작하지 않는가? (날씨 인사 등)
- [ ] 감정 형용사 1회 이내?
- [ ] 톤 일관성 (한 글에 ~습니다/~다 혼용 금지)?
- [ ] em-dash/en-dash 본문 0회?
- [ ] 길이 가이드 적합?
