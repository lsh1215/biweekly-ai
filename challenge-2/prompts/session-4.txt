## 이번 세션 목표 (Sprint 4: Decision journal + Cost summary + 최종 통합)

### 산출물

- `src/ria/journal.py` — append-only decision log.
  - 테이블 스키마 (idempotent CREATE):
    ```sql
    CREATE TABLE IF NOT EXISTS decisions (
      id SERIAL PRIMARY KEY,
      ts TIMESTAMPTZ NOT NULL DEFAULT NOW(),
      cycle_type TEXT NOT NULL,   -- 'planned' | 'interrupt_P0' | 'deferred_P1' | 'deferred_P2' | 'cooldown_skip' | 'error'
      ticker TEXT,
      action TEXT,                -- 'BUY' | 'HOLD' | 'REDUCE' | 'WATCH' | 'REVIEW' | 'SKIP' | 'ERROR'
      rationale TEXT,
      citations JSONB
    );
    ```
  - API: `journal.append(cycle_type, ticker, action, rationale, citations)`.
  - Sprint 2 agent loop과 Sprint 3 event_loop가 이 함수를 호출하도록 hook 추가 (retrofit OK).

- **Cost summary 생성 로직** — 모든 Anthropic API 호출에서 `response.usage.input_tokens` / `output_tokens` 수집 → `reports/cost_summary.md` 생성.

  **파일 포맷 엄격 (VERIFY.sh `awk 'NR==1 && /^total \$[0-9]+\.[0-9]+/'` 파서와 일치 필수)**:

  ```
  total $12.40
  planned_20260413: input=3214 output=892 usd=0.0421
  interrupt_P0_20260415_TSLA: input=2891 output=720 usd=0.0384
  classify_evt_tsla_earnings_miss: input=512 output=68 usd=0.0014
  ...
  ```

  제약:
  - **첫 줄은 정확히 `total $<숫자>.<숫자>` 리터럴** (마크다운 헤딩 `#` 금지, 콜론 `:` 금지)
  - 둘째 줄부터 per-call 내역
  - `total` 값이 $50 초과 시 exit 1 (session 자체 실패 처리). $40 초과 시 TIMELINE에 `COST_WARN=<usd>` append.

- **VERIFY.md 보강** — 기존 내용 유지하면서 상단에 macOS `caffeinate` 안내 추가:
  ```
  ## 전제
  - macOS 사용자는 overnight 실행 시 `caffeinate -di nohup bash scripts/overnight.sh &` 필수.
  ...
  ```

- **README.md** — **soft deliverable**. 생성 시도. 실패 시 TIMELINE 기록만, checkpoint fail 아님.
  한국어 quickstart (3~5줄 구조):

