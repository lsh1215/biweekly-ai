## 이번 세션 목표 (Sprint 1: Tool primitives + RAG ingest)

**경로 규약**: `challenge-2/` 루트 기준.
**Package**: `from ria.xxx` (접두 `src.` 금지. Sprint 0이 고정).

### 산출물

- `src/ria/tools/prices.py`:
  - `get_prices(tickers: list[str], window_days: int, as_of: date | None = None) -> pd.DataFrame`
  - fixture 조회, 비거래일 처리 (NYSE calendar via `pandas_market_calendars`)
  - `as_of` 기본값 = fixture 최대 날짜
  - 엣지: 존재 안 하는 티커, 날짜 범위 밖, 주말/휴장일

- `src/ria/tools/news.py`:
  - `get_news(ticker: str, last_n_days: int) -> list[Article]` (fixture 조회)
  - Article = dict with `title`, `link`, `pub_date`

- `src/ria/tools/rag.py`:
  - `rag_search(query: str, top_k: int = 5) -> list[Chunk]` — pgvector HNSW 질의
  - 임베딩: `sentence-transformers/all-MiniLM-L6-v2` (384-dim, 최초 1회 로컬 다운로드)
  - 쿼리는 영어 권장 (모델 한계 — TIMELINE에 주석 기록)

- `src/ria/ingest/filings.py`:
  - 공시 .txt → 청크 → 임베딩 → pgvector insert
  - **청킹 전략 고정 (v1)**: **섹션 기반 우선** (10-K의 `ITEM N.` 패턴 분할). 섹션 파싱 실패 시 **512자 고정 chunk** 폴백. TIMELINE에 `CHUNK_STRATEGY=section|fixed512` 기록.
  - HNSW 인덱스: `CREATE INDEX ... USING hnsw (embedding vector_cosine_ops);` — **디폴트 파라미터 (m=16, ef_construction=64)** 사용. 튜닝 X.

- `src/ria/db/schema.sql`:
  ```sql
  CREATE EXTENSION IF NOT EXISTS vector;
  CREATE TABLE IF NOT EXISTS filings_chunks (
    id SERIAL PRIMARY KEY,
    accession TEXT NOT NULL,
    section TEXT,
    text TEXT NOT NULL,
    embedding vector(384) NOT NULL,
    ts TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
  CREATE INDEX IF NOT EXISTS filings_chunks_embedding_idx
    ON filings_chunks USING hnsw (embedding vector_cosine_ops);
  ```
  docker-compose는 `pgvector/pgvector:pg16` 이미지라 extension 사용 가능.

- `tests/test_tools_prices.py`, `test_tools_news.py`, `test_tools_rag.py`:
  - 각 tool 엣지 케이스 (빈 fixture, 누락 티커, 주말/휴장일, fixture 범위 밖)
  - `rag_search`는 영어 쿼리로 테스트 (한국어 쿼리는 별도 skip test)

### Ingest 실행

- `tests/test_ingest_filings.py` — 임베딩 함수 unit test (384-dim 출력 확인)
- `scripts/run_ingest.py` — `python scripts/run_ingest.py` 1회 실행으로 `data/fixtures/filings/*.txt`를 전부 pgvector에 적재. 이 스크립트는 smoke test 범위.
- 성공 기준: `SELECT COUNT(*) FROM filings_chunks >= 10`.

## 작업 규칙
1. **TDD 강제 (`src/ria/**`)**: 각 tool 함수 테스트 먼저 → 실패 → 구현 → 통과.
2. **`scripts/run_ingest.py`는 smoke test 1개로 갈음** (exit 0 + pgvector row 수 증가 확인).
3. **결정론**: 모든 tool은 동일 입력에 동일 출력 (fixture 기반).
4. **pgvector HNSW**: 디폴트 파라미터. 튜닝 금지 (YAGNI).
5. **임베딩 캐시**: sentence-transformers 모델 최초 1회 다운로드 후 `~/.cache/` 재사용. offline 환경 대비 TIMELINE 기록.
